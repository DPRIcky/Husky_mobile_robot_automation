  • Linear Velocity: (vx, vy)

  • Constant acceleration motion model
  • Adaptive measurement weighting based on uncertainty

🎓 ALGORITHM EXPLANATION
═══════════════════════════════════════════════════════════════════════

Extended Kalman Filter (EKF):

1. PREDICTION STEP (50 Hz):
   x_k+1 = f(x_k, u_k) + w_k
   • Predicts next state using motion model
   • Increases uncertainty (covariance grows)
   • Motion model: constant acceleration

2. UPDATE STEP (sensor dependent):
   K = P*H'/(H*P*H' + R)        # Kalman gain
   x = x + K*(z - H*x)           # State update
   P = (I - K*H)*P              # Covariance update
   • Corrects prediction with sensor measurement
   • Reduces uncertainty
   • Kalman gain balances prediction vs measurement

Sensor Measurements:
   • IMU: [theta, omega, ax, ay] (4D)
   • Odometry: [x, y, theta, vx, vy, omega] (6D)
   • GPS: [x, y] (2D)

Why EKF?
   ✓ Handles nonlinear motion models
   ✓ Provides uncertainty estimates
   ✓ Optimal for Gaussian noise
   ✓ Computationally efficient

🔧 TUNING GUIDE
═══════════════════════════════════════════════════════════════════════

Filter Too Smooth (Lags Behind):
  → Decrease measurement noise (trust sensors more)
  → Increase process noise (trust model less)

Filter Too Noisy (Jittery):
  → Increase measurement noise (trust sensors less)
  → Decrease process noise (trust model more)

Uncertainty Grows Too Fast:
  → Decrease process noise
  → Check sensor data quality

Estimates Drift Over Time:
  → Enable GPS for absolute reference
  → Check odometry calibration
  → Verify IMU bias compensation
